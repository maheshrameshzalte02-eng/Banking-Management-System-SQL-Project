# Banking Management System Using PostgreSQL

## Project Overview

This project demonstrates a Banking Management System developed using PostgreSQL.

The database consists of four tables:

- Customers
- Accounts
- Transactions
- Loans

The project includes table creation, data insertion, and 17 SQL operations for banking data analysis.

---

## Technologies Used

- PostgreSQL
- SQL
- pgAdmin 4

---

## Database Tables

### Customers
Stores customer details.

### Accounts
Stores account information and balances.

### Transactions
Stores deposit and withdrawal records.

### Loans
Stores customer loan details.

---

## SQL Concepts Covered

- INNER JOIN
- LEFT JOIN
- GROUP BY
- HAVING
- Aggregate Functions
- Subqueries
- Window Functions

### Window Functions Used

- RANK()
- DENSE_RANK()
- LAG()
- LEAD()
- SUM() OVER()

---

## Project Tasks

Completed 17 SQL operations including:

1. Customer and account details using INNER JOIN
2. Top 3 customers with highest balance
3. Customers with loan details
4. Total deposits and withdrawals
5. Customer-wise transaction analysis
6. Above-average balance customers
7. Highest transaction by customer
8. Customers without loans
9. Total transactions per customer
10. Customer ranking by balance
11. Dense ranking by balance
12. Previous transaction using LAG()
13. Next transaction using LEAD()
14. Running total using SUM() OVER()
15. Second highest account balance
16. Customers with more than 2 transactions
17. Minimum and maximum transaction amounts
